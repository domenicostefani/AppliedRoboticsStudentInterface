Plans a path that maximizes the time-score of the mission.

For each victim collected a time-bonus is granted. At each step, the greedy function picks the victim that better improves the final score. To avoid robot loops and improve the search, the victims to test are ordered by distance from the starting point.

Parameters
safeBorderssafe borders without gate slot (used to prevent RRT bug)
slotBordersborders with gate slot (used for collision detection)
obstacle_listList of obstacle polygons.
victim_listList of victim polygons.
xStarting point x coordinate.
yStarting point y coordinate.
thetaStarting angle.
xfArrival point x coordinate.
yfArrival point y coordinate.
thfArrival angle.
config_folderConfiguration folder path.
Returns
The multi-point dubins curve solution.

Finds the victims in the arena given the arena image and detects victim number.

Victim color is green. Number is detected by template matching. Template matching is performed by extracting the axes-aligned minimal bounding rectangle for each region of interest and comparing it with the templates in four 90 degree orientations to maximize the matching score.

Multipoint Markov-Dubins Path planner.

Iterative Dynamic Programming version of the Multipoint Markov-Dubins Problem Inspired to the proposed idp solution proposed, it vary slightly in the fact that the first and last angles of the path are fixed and not free. The method is arranged for ANGLE REFINEMEN (it takes as argument both the solution to refine and the current angle range, which is granularity*numAngles)

Parameters
pathPoint path
startAngleFirst angle
arriveAngleLast angle
obstacle_listList of obstacles
numAnglesNumber of possible free angle choices (sampled)
roughSolutionPrevious path (to refine)
rangeAngle range (default is 2*M_PI, smaller for refinement)
Returns
Multipoint Dubins path (or empty vector if no collision-free path)

Finds the shortest multi-point dubins curve for the given path.

The recursive function has two base cases: two-points dubins::Curve and three-points dubins::Curve. The recursion step is called on N-points dubins::Curve. Free angles are chosen among a number of test angles.

Parameters
pathList of points in the path.
startIdxRecursion start index.
arriveIdxRecursion arrival index.
startAngleStarting angle.
arriveAngleArrival angle.
returnedLengthOutput length of the shortest path.
obstacle_listList of obstacle polygons.
num_anglesNumber of angles to test.
Returns
A pair with true if the path does not collide and the shortest multi-point dubins::Curve.
